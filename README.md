# Mortgage Rates Scraper + Python Backup Builder

This repo contains mortgage-rate scraping utilities and a Python backup exporter that builds a `bestrates.json` file you can compare against your `bestrate360.json` output.

## What this project does

### 1) Scrape lender rates
The primary scraper is TypeScript/Bun-based and lives in `scripts/rate-scraper/`.

- `scrape.ts` runs scraping for one lender or all lenders.
- Provider-specific extraction logic is in `scripts/rate-scraper/providers/*.ts`.
- Output is written to `data/rates/<lender>.json`.

### 2) Build a Python backup artifact
The Python utility `scripts/rate-scraper/build_bestrates.py` reads scraped lender files and generates a backup best-rate comparison file.

By default, it writes:

- `data/rates/bestrates.json`

This file is designed to be compared with your `bestrate360.json` project output.

---

## How it works (Python backup flow)

`build_bestrates.py`:

1. Reads all `*.json` files in `data/rates/` (excluding helper files like `bestrates.json`, `bestrate360.json`, and `updates.txt`).
2. Supports both formats:
   - Legacy array format: `[ ...rates ]`
   - Current format: `{ lenderId, ..., rates: [ ... ] }`
3. Normalizes rates to a consistent shape.
4. Computes:
   - Best rate by `(buyerType, rateType)`
   - Best **fixed** rate by `(buyerType, fixedTerm)`
5. Writes output containing:
   - `generatedAt`
   - `source` metadata
   - `segments` summary
   - `allRates` flattened list

## Quick start

## Requirements

- Python 3.10+
- Existing lender rate JSON files in `data/rates/` (generated by scraper or copied in)

## Run

```bash
python3 scripts/rate-scraper/build_bestrates.py
```

## Custom input/output paths

```bash
python3 scripts/rate-scraper/build_bestrates.py \
  --rates-dir /path/to/rates \
  --output /path/to/bestrates.json
```

If the rates directory does not exist, the script prints a message and exits with code `1`.

## How to compare with `bestrate360.json`

A common workflow:

1. Generate your main file in your other project (`bestrate360.json`).
2. Generate backup from this repo (`bestrates.json`).
3. Compare with a JSON diff tool (or script) to detect missing lenders/rates/segments.

Example using `jq` to inspect top-level differences quickly:

```bash
jq 'keys' data/rates/bestrates.json
jq 'keys' /path/to/bestrate360.json
```

## How to implement / extend

### Add a new lender scraper (TypeScript side)

1. Create `scripts/rate-scraper/providers/<lender>.ts` implementing the `LenderProvider` interface.
2. Register provider in `scripts/rate-scraper/scrape.ts`.
3. Run scraping and validation.

### Keep Python backup compatible

To ensure new provider output is usable by `build_bestrates.py`, include these fields in each rate object:

- `id`
- `lenderId`
- `type`
- `rate`
- `buyerTypes`

Recommended additional fields used by the backup output:

- `name`
- `apr`
- `fixedTerm`
- `minLtv`
- `maxLtv`

If fields are missing or invalid, affected rows are skipped and counted in `source.invalidRows`.

## Notes

- The Python script is intentionally independent from Bun runtime.
- It does **not** replace provider scraping logic; it builds a backup/comparison artifact from already-scraped lender files.
